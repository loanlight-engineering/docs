The pre-audit work is now **phase 0** of the [agent orchestrator](/pipeline/overview#router-then-orchestrator), not a separate stage service. Two foundation agents run concurrently with a `llama_index` infra step, and a ledger-observed barrier plus a classify-readiness gate separate phase 0 from phase 1.

<Note>
The legacy `PreAuditStageService.executePreAuditStage` (three branches A/B/C) and `executePartnerPreAuditStage` no longer exist. `PreAuditStageService` now exposes only `prepareEpcClassification` — it resolves **which** documents to (re)classify and the classification loan context before the orchestrator runs. Classification, URLA extraction, and LlamaCloud indexing are now phase-0 work owned by `AgentOrchestrator`.
</Note>

## Phase-0 members

<Columns cols={3}>
  <Card title="preAuditClassification" icon="scan-line">
    The [Pre-Audit-Classification agent](/agents/pre-audit-classification) splits and classifies the PDFs (classify-split). It `requiresIndexing`, so its presence drives the `llama_index` infra step.
  </Card>
  <Card title="urlaExtraction" icon="file-text">
    The [URLA extraction agent](/agents/urla-extraction) extracts the MISMO XML into per-borrower structured JSON and runs the rules engine. MISMO-only — no pipeline read.
  </Card>
  <Card title="llama_index infra step" icon="database">
    Not an agent — an infra step the orchestrator runs itself. It uploads the raw documents into the loan's [LlamaCloud pipeline](/subsystems/llamacloud-retrieval) for retrieval and waits for poller readiness.
  </Card>
</Columns>

## Sequence

The orchestrator runs phase 0 as follows (`AgentOrchestrator.run`, `IndexingService`):

<Steps>
  <Step title="Prepare the classification scope (EPC)">
    Before `run()`, `IntegrationOrchestrationService` registers the documents into `loan_documents`, captures the LOA-936 snapshot (`snapshot_doc_ids`, `snapshot_taken_at`) over the live, non-superseded documents, and calls `PreAuditStageService.prepareEpcClassification(...)` to resolve `documentsToClassify` and the classification loan context. See [Run lifecycle and deduplication](/pipeline/lifecycle-and-dedup) for the change-detection rules.
  </Step>
  <Step title="Materialize the infra step">
    If any planned agent `requiresIndexing` and the channel owns indexing (`skipIndexing = false`), the orchestrator calls `IndexingService.beginInfraStep` to commit the `llama_index` row in `audit_infra_steps` **before** any phase-0 agent can reach `tryCompleteAuditRun`. This prevents a fast phase-0 agent from completing the run before the index step's own row exists.
  </Step>
  <Step title="Run phase-0 members concurrently">
    The phase-0 agents (`preAuditClassification`, `urlaExtraction`) fire via `Promise.allSettled`, dispatched by identity. The `llama_index` infra step (`IndexingService.runIndexStep` — `getOrCreatePipeline` + association on `loans.llc_pipeline_id` + `uploadAndWaitForDocuments`) runs alongside them. Indexing uploads the raw documents, never the classified output. On a `PIPELINE_NOT_FOUND` error the step clears the stale reference, recreates the pipeline, and retries the upload.
  </Step>
  <Step title="Surface fatal failures">
    The infra step's throw must propagate (a poller failure fails the run), so it is awaited rather than tolerated. A URLA-extraction deadline error also escapes `run()` to fail the audit (#2412) — every other phase-0 agent failure stays tolerated and recorded in `audit_errors`.
  </Step>
  <Step title="Barrier — phase 0 → phase 1">
    `awaitPhaseTerminal` advances only when every phase-0 `agent_runs` row **and** the `llama_index` infra step are terminal. Then `awaitClassifyReadiness` runs a bounded gate confirming the `classify_split_runs` are ready, so no phase-1 agent runs against unclassified documents.
  </Step>
  <Step title="afterPhase(0) hook">
    The EPC channel freezes the LOA-936 snapshot against the pipeline the index step resolved (`auditService.persistSnapshot`), before phase-1 agents read it, and fires the best-effort `documents.classified` event so integrations can carve and push classified documents back to Encompass.
  </Step>
  <Step title="Rehydrate for phase 1">
    The orchestrator rehydrates phase-0 outputs (the URLA handoff: transformed PII fields and property addresses) into the execution context so the phase-1 PII and property agents do not re-extract from the MISMO XML.
  </Step>
</Steps>

<Note>
Document split happens **inside** the Pre-Audit-Classification agent (the `splitDocuments` node), not as an orchestrator pre-step. Document buffers are handed straight to the agent and never enter graph state or LangSmith traces.
</Note>

## SKIP environment short-circuits

Three env flags short-circuit phase 0 for testing and partial runs (handled in `IntegrationOrchestrationService` before or around `run()`):

| Env var | Effect |
|---|---|
| `SKIP_AI_PIPELINE` | Handled before phase 0: `"true"` completes the run immediately; any other truthy value simulates a pipeline failure. |
| `SKIP_LLAMA_INGESTION` | Runs phase 0 only (classify + URLA), with **no** `llama_index` infra step and no phase-1/2 agents (which query the index). The stripped agents are terminalized `skipped` so the ledger stays congruent. |
| `SKIP_LANGGRAPH_AGENTS` | Skips agent execution entirely and completes the audit; every seeded ledger row is terminalized `skipped`. |

## MISMO-hash caching

URLA extraction reuses prior work when the input is byte-identical. `checkForCachedUrlaExtraction(loanId, mismoXmlHash)` looks for a prior completed `audit_runs` row with the same loan and `mismo_xml_hash` and reuses its `urla_extraction_results` via `reuseUrlaExtractionResults` plus the rules engine; a cache miss runs a fresh extraction. The EPC path stores the hash at run creation; the Partner path sets it post-creation via `updateMismoXmlHash`. See [Run lifecycle and deduplication](/pipeline/lifecycle-and-dedup#mismo-xml-hash).

## Partner pre-audit

Partner loans classify and index at document-upload time (`partner-loans.controller`), so the Partner plan strips `preAuditClassification` from execution and sets `ctx.skipIndexing = true` (the `llama_index` gate no-ops). `urlaExtraction` still runs as a phase-0 agent against the audit-time MISMO XML and persists its own handoff — one URLA path shared with EPC. The stripped `preAuditClassification` ledger row is marked `skipped` so the ledger stays congruent with what actually ran.
