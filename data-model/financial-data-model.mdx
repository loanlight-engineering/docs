The credit, income, and appraisal subsystems are the three audit-attachable run ledgers. Each owns its own run table, rule engine (`*_rule_sets` + `*_rules`), overrides, corrections, and recovery columns, and each can attach to a parent [`audit_runs`](/data-model/core-tables#audit-runs) row through a nullable `audit_run_id`. The rule-engine pattern is uniform across all three.

A fourth subsystem ‚Äî **asset / Statement Analyzer** (`asset_runs` + the `asset_*` tables) ‚Äî is also a standalone run ledger, but it does **not** attach to `audit_runs`; it binds to a loan or to a `borrower_files` statement set. It is documented in [The asset subsystem](#the-asset-subsystem) below, alongside the older **bank-statement income pipeline** (`borrower_files` ‚Üí `bank_statements` ‚Üí `transactions`) it reads from.

## The shared rule-engine pattern

All three subsystems use the same shape (`*_rule_sets.ts` + `*_rules.ts`):

- **`*_rule_sets`** ‚Äî lender-scoped, versioned. PK `id`, `lender_id` (cascade), `name`, `version`, `template_version`, `is_active`, `created_by` ‚Üí `portal_users` (NOT NULL), timestamps. Partial unique `(lender_id, name)` where active.
- **`*_rules`** ‚Äî `rule_set_id` ‚Üí the set (cascade), `code`, `name`, `category`, `severity` (CHECK `FAIL | WARNING`), `thresholds` jsonb, `required_fields` text[], `form_types` text[], `evaluate_fn_key`, `operator` / `field_path` / `expected_value` / `condition` jsonb (mutually exclusive ‚Äî `chk_*_rule_type_exclusive`), `source` (CHECK `loanlight | fannie_mae | freddie_mac | uspap | agency_common`), `fail_message`, `pass_message`, `is_enabled`, `loan_program_id` ‚Üí `loan_programs` (set null). Each has a **migration-only** COALESCE unique on `(rule_set_id, code, COALESCE(loan_program_id, nil_uuid))` (Drizzle can't express it).
- **`loan_programs`** ‚Äî `match_program_aliases text[]` (nullable, migration `0257`, LOA-1872). An array of canonical program residuals (lowercased, noise-stripped by `normalizeProgramName`) used for alias-first program resolution on the EPC path. When a loan arrives with a `loanProgramName`, `normalizeProgramName` reduces it to a residual (strips ARM/term/IO tokens), and the resolver checks these arrays before falling back to the family √ó doc-type strategy. Seeded through both the appraisal and credit staged-bundle apply flows. Rows with no aliases leave this column `NULL` and resolve by family √ó doc-type only.
- The platform-owned catalog **`appraisal_rule_templates`** seeds all three. It has no `lender_id`, a `module` column (CHECK `appraisal | credit | income`, default `appraisal`), and is unique on `(code, module)`.

<Note>
A program-scoped rule (same `code`, with a `loan_program_id`) overlays/replaces the lender-global rule of the same code at evaluation time. Per-run overrides live in the `*_rule_overrides` tables (outcomes `PASS | FAIL | WARNING | NEEDS_INFO | SKIPPED`); latest row per `(run, rule_code)` wins.
</Note>

### credit_rules ‚Äî the FK difference

`credit_rules` follows the shared shape, with one structural difference from `appraisal_rules`:

- **`source_template_id` is intentionally NOT a foreign key** (migration `0080` removed it ‚Äî credit is decoupled from the appraisal module; pinned by `fk-ondelete-policy.spec.ts`).

Its `operator` CHECK reached parity with `appraisal_rules` in migration `0190`: it is now the **16-value** union and **does** include `abs_less_than` / `abs_less_than_or_equal`.

`credit_rule_split_drafts` (`credit-rule-split-drafts.ts`, migration `0189`) is dedicated credit split-draft persistence for the rule editor ‚Äî distinct from the appraisal-backed `rule_split_drafts` table. PK `id`, `user_id` ‚Üí `portal_users` (cascade), `lender_id` ‚Üí `lenders` (cascade), `original_text`, `second_half_text`, `first_half_saved_rule_id` ‚Üí `credit_rules` (set null), `created_at`, `dismissed_at`. Partial unique `(user_id, lender_id)` where `dismissed_at IS NULL`.

## The bank-statement income pipeline

The original bank-statement income family is rooted at **`borrower_files`** and is separate from the `income_runs` family above. It is also the statement source the [asset subsystem](#the-asset-subsystem) reads from.

### borrower_files

`borrower-files.ts`. One row per uploaded statement set. PK `id`, `lender_id` ‚Üí `lenders`, `loan_id` ‚Üí `loans` (set null).

<ParamField path="loan_reference" type="text">Optional external loan reference. CHECK `loan_reference_length_check`: `‚â§ 255` characters (migration `0195`).</ParamField>
<ParamField path="borrower_type" type="text NOT NULL" />
<ParamField path="status" type="text">Default `processing`.</ParamField>
<ParamField path="large_deposit_threshold_mode" type="text">Default `fixed`. CHECK `large_deposit_threshold_mode_check`: `fixed | pct_avg_qualifying_income | pct_avg_gross_deposits` (migration `0186`).</ParamField>

**Priority Financial income exclusion columns (LOA-1686, LOA-1690):**

<Note>
**Resilience columns (LOA-1078):** migration `0240` extended `borrower_files.status` CHECK to include `'queued'` and added `recovery_attempts integer NOT NULL DEFAULT 0`. ## The asset subsystem

The **asset / Statement Analyzer** subsystem (migrations `0180`‚Äì`0187`, `0207`, `0228`‚Äì`0238`) extracts and qualifies borrower asset accounts from bank statements. Unlike the credit/income/appraisal ledgers it does **not** attach to `audit_runs`; an `asset_runs` row binds to a loan or, for standalone runs, to a `borrower_files` set.

### asset_runs

`asset-runs.ts`. The run model, mirroring `audit_runs` idempotency. PK `id`, `lender_id` ‚Üí `lenders` (restrict, NOT NULL), `loan_id` ‚Üí `loans` (set null), `rule_set_id` ‚Üí `asset_rule_sets` (set null).

<ParamField path="run_number" type="integer NOT NULL">Monotonic per loan.</ParamField>
<ParamField path="status" type="varchar(20) NOT NULL">Default `pending`. CHECK `asset_runs_status_check`: `pending | processing | complete | failed` (note `complete`, the `audit_runs` spelling ‚Äî not `completed`).</ParamField>
<ParamField path="superseded_by" type="uuid (self-FK, set null)">A re-extraction supersedes the prior run; `superseded_at` is set before row reconciliation.</ParamField>
<ParamField path="borrower_file_id" type="uuid ‚Üí borrower_files (set null)">Statement-set binding for standalone (loanless) runs (migration `0207`).</ParamField>

Unique `(loan_id, run_number)` (`asset_runs_loan_run_key`); for loanless runs, a partial unique `(borrower_file_id, run_number)` `WHERE loan_id IS NULL` (migration `0207`). Indexes on `lender_id` and a partial `loan_id` index `WHERE superseded_at IS NULL`. Plus `created_at`, `completed_at`.

### asset_accounts

`asset-accounts.ts`. One row per extracted or manual borrower asset account. PK `id`, `loan_id` ‚Üí `loans` (cascade; nullable since migration `0207`), `borrower_file_id` ‚Üí `borrower_files` (cascade, migration `0207`).

<ParamField path="source" type="asset_source (pgEnum) NOT NULL">`extracted | manual | plaid`. Migration `0228` added `plaid` for accounts ingested via the Plaid Link (Unified Intake) flow (LOA-1580).</ParamField>
<ParamField path="source_statement_id" type="uuid ‚Üí bank_statements (set null)" />
<ParamField path="matched_borrower_id" type="uuid ‚Üí borrowers (set null)" />
<ParamField path="asset_type_normalized" type="asset_type_normalized (pgEnum) NOT NULL">Default `unknown`. `crypto` added in migration `0187`.</ParamField>
<ParamField path="asset_class" type="asset_class (pgEnum)">`liquid | near_liquid | retirement | restricted | manual | unknown`.</ParamField>
<ParamField path="account_state" type="asset_account_state (pgEnum) NOT NULL">Default `requires_review`. `requires_review | included | excluded`.</ParamField>

Numeric columns: `current_balance`, `qualifying_balance`, `qualifying_percent` numeric(5,4), `ownership_percent` numeric(5,4) (NOT NULL, default `1`), `qualifying_assets`. <Note>
`qualifying_assets = qualifying_balance √ó qualifying_percent √ó ownership_percent` is computed downstream; the table only guarantees the column exists.
</Note>

### asset_rule_sets and asset_qualifying_schedules

`asset-rules.ts`. The asset rule engine **differs from the shared `*_rule_sets` pattern**: no `created_by`, no `template_version`, and no partial-unique-where-active.

- **`asset_rule_sets`** ‚Äî PK `id`, `lender_id` ‚Üí `lenders` (cascade), `program_id` ‚Üí `loan_programs` (set null), `name`, `version` (default `1`), `is_active` (default `true`), `large_deposit_threshold_mode` (`large_deposit_threshold_mode` pgEnum: `pct_avg_gross_deposits | fixed`), `large_deposit_threshold_value` numeric, timestamps.
- **`asset_qualifying_schedules`** ‚Äî the `(rule_set_id, asset_type, purpose)` ‚Üí `qualifying_percent` haircut lookup. PK `id`, `rule_set_id` ‚Üí `asset_rule_sets` (cascade), `asset_type` (`asset_type_normalized` pgEnum), `purpose` text (default `eligibility`), `qualifying_percent` numeric(5,4) (CHECK `[0,1]`). Unique `(rule_set_id, asset_type, purpose)`.

### asset_rules and asset_conditions

**`asset_rules`** (`asset-rules.ts`) mirrors the `appraisal_rules` conditional-assertion shape but is leaner: PK `id`, `rule_set_id` ‚Üí `asset_rule_sets` (cascade), `code`, `name`, `category`, `severity` (CHECK `FAIL | WARNING`), `thresholds` jsonb, `required_fields` text[], `operator` / `field_path` / `expected_value` / `condition` (mutually exclusive ‚Äî `asset_rules_type_exclusive`), `description`, `source` (text, default `loanlight`, **no CHECK**), `fail_message`, `pass_message`, `is_enabled`, timestamps. The `operator` CHECK is the **16-value** union including `abs_less_than` / `abs_less_than_or_equal`. There is **no** `evaluate_fn_key`, `form_types`, `loan_program_id`, or `source_template_id` ‚Äî program/lender scope is inherited from the owning rule set.

**`asset_conditions`** (`asset-conditions.ts`) ‚Äî the documentation-condition lifecycle. PK `id`, `asset_account_id` ‚Üí `asset_accounts` (cascade ‚Äî FK'd to the account, not a run, so conditions survive run supersession), `rule_code`, `status` text (default `open`, CHECK `open | satisfied | waived | superseded`), `emitted_at`, `waived_at`, `waived_by` uuid, `waiver_reason`, timestamps.

### asset_corrections

`asset-corrections.ts`. Auditable UW-override rows for asset accounts (last-write-wins; older rows retained for audit). PK `id`, `asset_account_id` ‚Üí `asset_accounts` (cascade), `field`, `old_value`, `new_value`, `changed_by` ‚Üí `portal_users` (set null), `changed_at`, `reason`. Indexed on `(asset_account_id, field)`.

### statement_invites

`statement-invites.ts` (migration `0230`). Borrower-invite lifecycle for the Plaid Link (Unified Intake) path. PK `id`, `loan_id` ‚Üí `loans` (cascade, NOT NULL), `borrower_email` text (NOT NULL), `borrower_name` text (NOT NULL), `token_hash` text (NOT NULL), `status` `statement_invite_status` pgEnum DEFAULT `created`, timestamps (`sent_at`, `opened_at`, `connected_at`, `ready_at`, `failure_reason`, `updated_at`).

`statement_invite_status` values: `created | sent | opened | linking | connected | ready | failed | expired | cancelled | undelivered`.

Partial unique index `statement_invites_one_active_uniq` on `(loan_id, lower(borrower_email))` WHERE status NOT IN terminal states. A hashed-token index `statement_invites_token_hash_idx` supports the borrower landing-page lookup.

### plaid_items

`plaid-items.ts` (migration `0231`). Persists a borrower's Plaid connection per statement invite. PK `id`, `statement_invite_id` ‚Üí `statement_invites` (cascade), `access_token_ciphertext` text (NOT NULL ‚Äî KMS-encrypted; no plaintext token stored), `item_id` text (NOT NULL, unique), `institution` jsonb, `mask` text (last-4, masked), `account_type` text, timestamps.

<Warning>
The Plaid access token is stored **only as KMS ciphertext** in `access_token_ciphertext`. Raw account numbers are never persisted ‚Äî only the last-4 `mask`. A `plaid_items` row is deleted when its parent `statement_invites` row is deleted (cascade).
</Warning>
