The request schemas for the Partner API are NestJS DTOs in `partner-api/dto/`, validated with `class-validator`. This page is the field-level reference; the endpoint pages ([Loans](/partner-api/loans), [Audits](/partner-api/audits)) describe behaviour.

## CreatePartnerLoanDto

Body of [`POST /api/v1/partner/loans`](/partner-api/loans). File: `create-partner-loan.dto.ts`.

| Field | Type | Required | Validation |
| --- | --- | --- | --- |
| `loanNumber` | string | yes | `@IsString @MaxLength(100) @Sanitize(100)` |
| `mismoXml` | string (base64) | yes | `@IsString @IsSafeBase64()` (Buffer round-trip â€” stack-safe for multi-MB payloads) |
| `documents` | `DocumentDto[]` | no (default `[]`) | `@IsArray @ValidateNested({each}) @Type(DocumentDto)` |
| `metadata` | `LoanMetadataDto` | no | `@ValidateNested @Type(LoanMetadataDto)` |
| `skipClassification` | boolean | no (default `false`) | `@IsBoolean`. Docs are still registered and uploaded; only classification is skipped. |

### DocumentDto

| Field | Type | Required | Validation |
| --- | --- | --- | --- |
| `fileName` | string | yes | `@MaxLength(255) @Sanitize` |
| `content` | string (base64) | yes | `@IsSafeBase64()` |
| `mimeType` | string | **yes** | `@IsString() @MaxLength(100) @Sanitize(100)` â€” no `@IsOptional()`, so omitting it (or sending a value `@Sanitize` wipes) fails validation with `400` |
| `documentType` | string | no | `@MaxLength(100)`; captured but informational |

### LoanMetadataDto

| Field | Type | Required | Validation |
| --- | --- | --- | --- |
| `estimatedClosingDate` | string | no | `@IsDateString` (YYYY-MM-DD) |
| `lenderCode` | string | no | â‰¤ 100 chars |
| `loanType` | string | no | â‰¤ 100 chars (see note) |
| `loanProgramName` | string | no | â‰¤ 255 chars |

<Warning>
**`loanType` is free-form text (the `LoanTypeSchema` mirrors this).** The DB column is `text`; the value passes through verbatim (`loan.service.ts::resolveLoanType`). There is **no** `@IsEnum` and **no** `@Matches` regex.

This is enforced by design (LOA-1088). The `partner-loans.controller.spec.ts` test group `"LoanTypeSchema (OpenAPI response schema â€” LOA-1088 contract parity)"` asserts the response schema has no `enum` key, no `pattern` key, `type: "string"`, `maxLength: 100`, and `nullable: true`. Re-adding `enum: Object.values(LoanType)` or a slug regex like `^[A-Za-z0-9_-]+$` re-introduces the contract lie and rejects valid values such as `bank_statement_12mo` or aliased labels like `"Bank Statement - 12 Month"`.

`@Sanitize(100)` runs **before** `@MaxLength(100)` (it truncates to 100 first; non-string inputs are wiped to `undefined` so `@IsOptional` short-circuits). The max lengths differ intentionally: `loanType = 100`, `loanProgramName = 255`.
</Warning>

## CreatePartnerAuditDto

Body of [`POST /api/v1/partner/audits`](/partner-api/audits). File: `create-partner-audit.dto.ts`.

| Field | Type | Required | Validation |
| --- | --- | --- | --- |
| `loanId` | string (uuid) | yes | `@IsUUID()` |
| `agents` | string[] | no | `@IsArray @IsString({each})`. Omitted â†’ all available agents run. |
| `options` | `AuditOptionsDto` | no | `@ValidateNested @Type` |

Example `agents` values: `piiValidation`, `documentHealth`, `urlaExtraction`, `propertyData`, `redFlags`, `creditEvaluation`. These are the canonical agent name strings used by `AuditAgentsExecutor`.

## Response schemas

The full response shapes are documented on the endpoint pages, since several differ from their Swagger schemas:

- Loan create / detail / status / list, document upload and listing â€” [Loans API](/partner-api/loans).
- Audit create / status / errors / classifications â€” [Audits API](/partner-api/audits).
- Audit results (`agents`, `creditEvaluations`, `summary`) and access URLs â€” [Results and access URLs](/partner-api/results-and-access-urls).

<Note>
**Two known schema/reality gaps to keep in mind:** `GET /loans` returns raw snake_case `LoanRecord` rows (not the camelCase Swagger schema), and document statuses in create/upload responses are `new | updated | existing` (not the Swagger example's `"uploaded"`, which only appears in detail/pipeline responses).
</Note>
