The encompass service owns its own database: Aurora Serverless v2 PostgreSQL, schema `loanlight_epc`. It is separate from the loanlight-api / AUS database.

## Migrations

Migrations live in `packages/encompass/migrations/*.sql` and are applied by the `DbMigrate` Lambda custom resource on every deploy.

<Warning>
There is **no applied-migrations ledger.** Every `.sql` is re-run whenever the migrations' content hash changes, so **every migration must be idempotent** (`CREATE TABLE/INDEX IF NOT EXISTS`, `ADD COLUMN IF NOT EXISTS`, guarded `ALTER`). A non-idempotent statement wedges `LLIntegrationsDatabaseStack` in `UPDATE_ROLLBACK_FAILED` and blocks all deploys (LOA-1004).
</Warning>

The current schema has **12 tables**. Migration `001_baseline_schema.sql` creates the original baseline; migrations 003 through 013 add the rest.

| Migration | Adds |
|---|---|
| `001_baseline_schema` | `loan`, `client`, `transaction_record`, `transaction_state`, `epc_event`, `transaction_update` |
| `002_add_processed_at_to_events` | `epc_event.processed_at` |
| `003_create_audit_details` | `audit_details` |
| `004_drop_is_processed_successfully` | drops a `transaction_update` column |
| `005_add_error_details_to_audit_details` | error columns on `audit_details` |
| `006_add_progress_to_audit_details` | progress columns on `audit_details` |
| `007_create_transaction_documents` | `transaction_document` |
| `008_transaction_is_shadow` | `transaction_record.is_shadow`, widen `loan.encompass_loan_number` to 64 |
| `009_create_document_push_records` | `document_push_record` |
| `010_create_epc_delivery_records` | `epc_delivery_records` |
| `011_add_delivery_cancelled_after` | `transaction_state.delivery_cancelled_after` |
| `012_create_lender_and_client_lender_id` | `lender`, `client.lender_id` |
| `013_create_audit_delivery_log` | `audit_delivery_log` |

<Note>
The `encompass/README.md` "expected tables" list is stale — it names only the 6 baseline tables. The six tables added by migrations 003–013 (`audit_details`, `transaction_document`, `document_push_record`, `epc_delivery_records`, `lender`, `audit_delivery_log`) are missing from it. Trust the schema, not the README.
</Note>

## Access

Database roles: `db_readonly` (all envs) and `db_readwrite` (int only). The bastion is shared with the API DB; connect on local port 5433 to avoid the 5432 conflict (`db:connect:int:read` / `db:connect:int:write` scripts). See [Integrations infrastructure](/integrations/infrastructure).
