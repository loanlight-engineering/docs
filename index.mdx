LoanLight is an AI-powered automated underwriting system (AUS) for mortgage loans. It ingests a loan package (a MISMO 3.4 XML application plus supporting PDF documents) from a loan origination system (Encompass) or directly through a Partner API, runs a battery of LangGraph AI agents over the documents and borrower data, and produces a structured audit that underwriters review. The audit surfaces issues: document health problems, PII inconsistencies, URLA (Form 1003) completeness gaps, property red flags, credit findings, and more.

## Who these docs serve

These pages are the shared engineering knowledge base for two audiences:

- **LoanLight engineers** who build, operate, and debug the AUS, the lender portal, the integrations, and the supporting infrastructure.
- **The Octy AI agent (Oz)**, which reads this knowledge base through the Mintlify MCP server to investigate and fix reported audit bugs. This documentation set is intended to replace the large inline architecture section that Octy previously baked into the Oz prompt. See [using these docs and the MCP](/overview/using-these-docs).

The code is always the source of truth. Where these docs note a contradiction with an older system prompt or README, they document the code and flag the misconception.
