The lender portal (`@loanlight/app`, in `loanlight-api/packages/app`) is the Next.js 15 web app that lender staff use to run and review LoanLight's AI underwriting tools. It is a stateless Backend-For-Frontend (BFF): it holds no pipeline business logic and proxies almost everything to the [loanlight-api backend](/overview/repositories) under `/api/v1/portal/*`.

- **Prod URL:** `https://portal.loanlight.com`
- **Local:** `http://localhost:3001` (the NestJS/Fastify backend runs on `:3000`)
- **Metadata title:** "LoanLight Portal — AI-powered underwriting tools"

## What staff do here

## Who uses it

The portal has four roles: three tenant roles (`client_admin`, `client_user`, `read_only`) plus one internal global role (`super_admin`). See [authentication and roles](/portal/auth-and-roles) for how access is enforced.

Internal super admins can also impersonate any tenant ("View as Client") to reproduce what a lender sees. See [impersonation](/portal/impersonation).

## Two surfaces, one app

The portal renders two distinct UIs from the same codebase:

## Tech stack

| Concern | Choice |
|---|---|
| Framework | Next.js 15.5.19 (App Router), `output: "standalone"` |
| UI | React 19, Tailwind CSS v4, shadcn/ui + radix-ui, `@loanlight-engineering/shared-ui`, `lucide-react` |
| Auth | WorkOS AuthKit via `@loanlight-engineering/shared-auth` |
| AWS | `@aws-sdk/client-s3` (uploads), `@aws-sdk/client-ses` (invite/reset emails); LocalStack locally |
| LLM | `@anthropic-ai/sdk` for natural-language rule parsing and an agentic extraction fallback |
| Observability | Datadog RUM + browser-logs, `dd-trace`, `pino` |
| Testing | Vitest (unit, jsdom), Playwright + `playwright-bdd` (Gherkin) |

<Note>
The portal is stateless. It has no local database. The App `CLAUDE.md` line claiming `src/db/schema/*.ts` and `pnpm db:push` is out of date — the only persistence helpers are `src/lib/s3.ts` (S3 uploads) and `src/lib/email/ses.ts` (SES). All real data lives in [loanlight-api](/overview/repositories). See [portal architecture (BFF)](/portal/architecture-bff).
</Note>
