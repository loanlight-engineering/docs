Liabilities-Assessment (`liabilitiesAssessment`) is the only genuine stub in the agent catalog.
It is seeded `active = false` with `audit_phase = NULL`, is never executed, and has no nodes,
schema, rules, module, or DB tables. Liabilities and DTI are handled inside the
[Credit-Evaluation agent](/agents/credit-evaluation) and the URLA completeness rules.

<Note>
The legacy Octy system prompt lists both `creditEvaluation` and `liabilitiesAssessment` as
stubs. Only `liabilitiesAssessment` is actually a stub — Credit-Evaluation is fully
implemented.
</Note>

## What the code contains

The agent's only file is `langgraph/agents/liabilitiesAssessment/index.ts`:

```ts
export function buildGraph() {
  throw new Error("liabilitiesAssessment agent not yet implemented");
}
```

There is no node directory, no state schema, no rule library, no NestJS module, and no
`liabilities_*` tables. It is not wired into any audit. In the platform catalog (`agents`
table), the row is seeded `active = false` with `audit_phase = NULL`, so the router and the
legacy fan-out both skip it.
